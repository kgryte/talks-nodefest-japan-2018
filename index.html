Machine Learning

Machine learning is applied mathematics and statistics and largely resides in the domain of computer science.

In the field of statistics, we focus on mathematical formulas, the derivation of robust estimators, and developing techniques for modeling problem domains.

In computer science, we focus less on mathematical purity and more on the practical aspects of writing algorithms which can run efficiently and with high performance and give us results which are "good enough".

The primary aim of machine learning is to find patterns in data, to find the proverbial signal in the noise.

Models

To describe those patterns we want to find, we formulate and train models.

A model is a mathematical formulation summarizing the nature of, and describing the relationship between, data variables.

Models serve as the statistical foundation for machine learning.

In order for machine learning algorithms to find patterns in data, we must "train" (or "teach") machine learning algorithms as to what those patterns are.

In a sense, machine learning algorithms "learn" the models used to describe particular patterns. That learning often comes in the form of learning specific values of coefficients within an equation.

As an example...

What will be the temperature tomorrow?

$$t_n = t_{n-1}$$

$$t_n = \frac{1}{7} \sum_{i=n-7}^{n-1} t_i$$

$$t_n = \frac{1}{W} \sum_{i=n-W}^{n-1} t_i$$

...suppose we want to "predict" the temperature tomorrow. A naive approach we could use would be to predict that the temperature tomorrow is whatever the temperature is today.

Our naive approach can be expressed mathematically as t_n = t_{n-1}, which is a way of saying that, for a given day n, the temperature should equal the temperature of the previous day. This equation is our model.

However, we might, perhaps rightly, believe that this naive approach might not be as accurate as we would like due to day-to-day temperature variability (i.e., noise).

For example, an anomalous weather pattern may lead to a very warm day followed by a very cold day.

So, one possible refinement would be to predict that the temperature tomorrow is the average temperature of the past 7 days.

By taking the average, we will "smooth" out some of the day-to-day variability and hopefully have a more reliable estimate over the long term.

In other words, by using the average temperature of the previous 7 days to predict tomorrow's temperature, our predictions will be closer (i.e., have a smaller error), on average, to the actual temperature than had we used our naive model.

We can parameterize our model with the variable W which represents the number of days we want to average over.

In this example, the "model" is the mathematical formula describing the computation of the average temperature. That model is "trained" by giving the model data (e.g., the temperature for each of the past 7 days). In this case, the output of the trained model is a single value, which we then use to "predict" tomorrow's temperature.

Training Algorithms

  • Batch: build a model from a "batch" of data
  • Incremental: continuously update a model as data arrives
  • Mini-batch: hybrid of batch and incremental training.

How we train machine learning algorithms comes in two varieties.

The first is "batch". Batch training means training a model over an entire dataset. Many "big data" algorithms (e.g., such as those run on Hadoop) are of this variety.

The second is "incremental". Incremental training means training a model incrementally, one datum at a time. Such an approach allows one to continuously update a model as data arrives.

I should mention in passing that a third "variety" exists, commonly referred to as mini-batch.

Mini-batch is a hybrid of batch and incremental. In this variety, we buffer data and train models in small batches. The primary advantage of mini-batch is the ability to achieve some of the robustness of batch training with a close approximation of the immediacy (i.e., near real-time) of incremental training.

Real-Time Machine Learning

As the subject of this talk is real-time machine learning, the talk's focus is how we can develop, and use, incremental training machine learning methods in JavaScript.

...to help motivate real-time machine learning use cases, let us consider real-world data streams...

Example Data Streams

  • Mouse/cursor movements
  • Scrolling
  • Clicks
  • Sensors
  • Monitoring

In JavaScript environments, such as the web browser, we have data streams such as mouse and cursor movements. Each movement of a cursor generates a set of (x,y) coordinates.

Similarly, another data stream is scrolling.

Another data stream is clicks (or selecting a UI element).

In the web browser, on mobile devices, and in Node.js, we have access to sensor data, such as device orientation, temperature, and ambient lighting, which is being continuously recorded.

And finally, we have monitoring data, such as logs of failed HTTP/S requests, server errors, et cetera.
